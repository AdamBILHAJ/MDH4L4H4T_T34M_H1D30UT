Your frontend already targets port 8000 on the same host. Environment variable overrides (VITE_API_BASE, VITE_WS_BASE, etc.) are supported if needed.

3. Step-by-Step Launch Commands
Step A вЂ” Activate Python venv & start Daphne (Backend on port 8000)
Open PowerShell (as admin if needed) at the project root:

# 1. Activate the virtual environment
.\venv\Scripts\Activate.ps1

# 2. (Optional) Migrate if schema has changed
python .\backend\manage.py migrate

# 3. Start Daphne (ASGI server on port 8000)
daphne -p 8000 -b 0.0.0.0 core.asgi:application

 
Run from: C:\Users\ADAM BELHADJ\Desktop\mdhalahat_server\

Daphne will serve both HTTP REST APIs and WebSocket endpoints.

Step B вЂ” Start React dev server (Frontend)
Open a second terminal (PowerShell or cmd) вЂ” keep the first one running:

# 1. Navigate to the frontend directory
cd .\frontend

# 2. Install dependencies (skip if node_modules already exists)
npm install

# 3. Start Vite dev server
npm run dev
Vite defaults to port 5173. The frontend will be available at http://localhost:5173.

6. Troubleshooting Notes
Daphne not recognized: Make sure you activated the venv first (Activate.ps1). Daphne executable lives at venv\Scripts\daphne.exe.
CORS error in browser console: Confirm the frontend's window.location.hostname resolves to one of the CORS_ALLOWED_ORIGINS. With CORS_ALLOW_ALL_ORIGINS = True this should not be an issue.
WebSocket connection fails: Ensure Daphne (not runserver) is running вЂ” only Daphne handles the ProtocolTypeRouter in asgi.py properly.
The asgi.py has duplicated code blocks (lines 1вЂ“26, 27вЂ“40, 41вЂ“56). The last block (lines 41вЂ“56) is the effective one вЂ” the earlier duplicates are harmless since they get overwritten, but you may want to clean them up.
